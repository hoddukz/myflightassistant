## 1. 시작하기

### 계정

- 앱을 열고 이메일로 회원가입합니다.
- 로그인하면 대시보드가 표시됩니다.
- 동시 접속은 최대 2대까지 가능합니다. 3번째 기기에서 로그인하면 가장 오래된 세션이 자동 로그아웃됩니다.
- 30분간 미사용 시 자동 로그아웃됩니다.

### 스케줄 업로드

1. **설정** (하단 내비게이션 톱니바퀴 아이콘)으로 이동합니다.
2. **Roster Upload / Sync**을 탭합니다.
3. **방법 A — 파일 업로드:** `.ics` 또는 `.csv` 로스터 파일을 드래그 앤 드롭(또는 찾아보기)합니다.
4. **방법 B — 캘린더 싱크:** Google Calendar 비공개 iCal URL을 붙여넣고 **Save**를 탭합니다. 이후 **Sync Now**로 최신 스케줄을 수동으로 가져올 수 있습니다.

스케줄이 로드되면 모든 페이지에 트립, 비행, 듀티 정보가 자동으로 채워집니다.

## 2. 대시보드

홈 화면 — 필요한 모든 정보를 한눈에 확인합니다.

### 월간 통계

상단 3개 카드로 이번 달 현황을 표시합니다:

| 카드 | 내용 |
|------|------|
| **Trips** | 이번 달 완료/전체 트립 수 |
| **Flights** | 이번 달 완료/전체 비행 수 |
| **Off Days** | 이번 달 오프 일수/전체 |

### Block Hours

이번 달 완료 블록 타임 대비 스케줄 전체 블록 타임을 프로그레스 바로 표시합니다.

### FAR 117 FDP

현재 Flight Duty Period 상태를 간략히 표시하는 카드입니다:
- **온듀티:** 사용 시간/한도 프로그레스 바 + 잔여 시간.
- **오프듀티:** 다음 듀티의 Max FDP + 다음 듀티 날짜.
- 탭하면 **듀티** 페이지 전체 화면으로 이동합니다.

### Inbound Aircraft (인바운드 항공기)

다음 비행에 배정된 기체를 실시간 추적합니다:
- **비행 단계:** 게이트 출발, 이륙, 상승, 순항, 강하, 접근, 착륙 — 아이콘과 라벨로 표시.
- **프로그레스 바:** 비행 거리/전체 거리.
- **ETA 카운트다운**.
- **지연 정보** (해당 시).
- 콜사인을 탭하면 FlightAware 실시간 추적 페이지가 열립니다.

> 인바운드 비행이 감지될 때만 이 섹션이 표시됩니다.

### Current Trip Timeline (현재 트립 타임라인)

트립 진행 상황을 시각적 바로 표시합니다:
- 공항 순서 (예: DTW → ORD → MSP).
- 현재 Day 표시 (Day 1/3).
- 완료/남은 레그 수.

### Next Flight (다음 비행)

다음 비행 정보를 한눈에 확인합니다:
- 편명, 구간, 기종.
- 리포트 타임 및 출발 시간 (로컬).
- 카운트다운 타이머 (일/시/분).
- 탭하면 해당 비행의 **브리핑** 페이지로 이동합니다.

### Layover Info (레이오버 정보)

레이오버 중일 때 표시됩니다:
- 호텔명 (탭하면 Google Maps로 이동).
- 전화번호 (탭하면 전화 연결).
- 레이오버 시간 및 릴리즈 타임.

### Upcoming Events (다가오는 이벤트)

현재 트립 이후 다음 이벤트 — 페어링 ID, 날짜, 블록/크레딧 시간.

### Today's Flights (오늘의 비행)

오늘 비행 목록 — 시간, 구간, 기종, 데드헤드 표시. 완료된 비행은 연하게 표시됩니다.

## 3. 브리핑

비행 전 기상/NOTAM 종합 브리핑 페이지입니다.

### 내비게이션

- **트립 선택:** 좌우 화살표로 트립 간 전환. 페어링 ID와 날짜 범위 표시.
- **탭 바:** Overview 탭 + Day 탭 (Day 1, Day 2, ...).
- 지나간 날짜의 Day 탭은 연하게 표시되어 쉽게 구분됩니다.

### Overview 탭

트립 전체를 요약하는 5개 섹션:

| 섹션 | 내용 |
|------|------|
| **Trip Summary** | 페어링 ID, 일수, 레그 수, 블록/크레딧/TAFB, 레이오버 도시 |
| **Today** | 리포트/릴리즈 시간, 오늘 경로, 호텔 정보 (듀티가 있는 날만 표시) |
| **FDP Status** | 간략 FDP 표시 — 온듀티 시 프로그레스 바, 오프듀티 시 Max FDP 텍스트 + FAR 117 경고 |
| **Fatigue Risk** | Day별 듀티 시간 바 차트 + 위험 플래그 (짧은 휴식, 연속 장시간 듀티, 늦은 릴리즈 후 이른 리포트, 4일+ 연속 근무). 위험도: Low / Moderate / High |
| **Weather Alerts** | 처음 2일의 공항별 최악 기상 카테고리 뱃지 (VFR/MVFR/IFR/LIFR). Day 3 이후는 "Not yet available" (TAF 범위 초과) |

### Day 탭

각 Day는 레그별 브리핑을 표시합니다:

- **레그 헤더:** 편명, 구간, 기종, 데드헤드 표시.
- **시간:** 로컬 및 UTC 출발/도착, 블록 타임.
- 출발/도착 **공항 카드** (탭하여 펼침):

#### 공항 브리핑 카드

카드 헤더에 항상 표시되는 정보:
- 공항 코드, 기상 카테고리 뱃지 (VFR/MVFR/IFR/LIFR).
- 간략 정보: 기온, 풍향/풍속, 시정, 운고.
- 접힌 상태 인디케이터: SIG, AIR, 크리티컬 NOTAM 수, 기상 현상.

탭하여 펼치면 4개 상세 탭이 표시됩니다:

| 탭 | 내용 |
|----|------|
| **METAR** | Raw METAR (키워드 하이라이트), 디코딩된 풍향/시정/운고/기상/운층, AWC 소스 링크 |
| **TAF** | Raw TAF, 시간대별 예보 + 카테고리 표시. 도착 시간대는 "ETA" 뱃지로 강조 |
| **NOTAM** | 전체 NOTAM. 크리티컬은 빨간색 강조. 키워드(RWY, ILS, CLSD 등) 하이라이트 |
| **SIGMET** | 경로상 SIGMET/AIRMET — 미니 지도에 영향 범위 표시 + 고도, 심각도, 이동 방향 |

### 공항 검색

우측 상단 돋보기를 탭하여 ICAO 코드(3-4자)로 임의 공항 브리핑을 검색할 수 있습니다.

## 4. 크루 & 호텔

트립별 크루 구성 및 레이오버 호텔 정보를 확인합니다.

### 레이아웃

- 트립별, 그 안에서 Day별로 정리됩니다.
- 지나간 트립/Day는 "DONE" 뱃지가 표시됩니다.
- Day 카드를 탭하여 펼칩니다.

### 펼친 Day

- **크루 목록:** 이름, 사번, 포지션 뱃지 (CA / FO / FA / FF), 시니어리티 순 정렬.
- **호텔 정보:** 호텔명 (탭하면 Google Maps), 전화번호 (탭하면 전화), 레이오버 시간, 릴리즈 타임.

## 5. 스케줄

캘린더와 타임라인으로 모든 이벤트를 확인합니다.

### 캘린더

- 월별 그리드. 오늘은 파란색으로 강조.
- 파란 점 = 트립/페어링, 초록 점 = 오프.
- 날짜를 탭하면 타임라인이 해당 이벤트로 스크롤됩니다.

### 타임라인

- 이벤트 유형 뱃지 (PAIRING, MOV, TRAINING, VAC 등)가 있는 세로 목록.
- **펼치기**를 탭하면 Day별 상세 정보:
  - 리포트 타임, 각 레그 (편명, 구간, 시간, 크루, 테일 넘버).
  - 레이오버 정보 (호텔, 전화, 시간).
  - Day 합계 (블록, 크레딧, 듀티).
  - 트립 합계 (TAFB, 총 블록, 총 크레딧).
- 테일 넘버를 탭하면 FlightAware가 열립니다.

## 6. 듀티 (FAR 117)

FAR 117 듀티 타임 모니터링 및 픽업 판단 도구입니다.

### FDP 현황

- **온듀티:** 현재 FDP 시간 + 프로그레스 바 + 잔여 시간.
- **오프듀티:** 다음 듀티의 Max FDP.

### 누적 한도

| 항목 | 한도 |
|------|------|
| 최근 28일 비행시간 | 100시간 |
| 최근 365일 비행시간 | 1,000시간 |
| 168시간 내 56시간 이상 휴식 | 충족 필수 |

각 항목은 색상 프로그레스 바로 표시됩니다 (초록 → 노랑 → 빨강).

### FAR 117 Table B

접었다 펼 수 있는 참조 테이블 — 리포트 시간대와 레그 수별 FDP 한도. 현재 해당하는 행/열이 강조됩니다.

### 픽업 시뮬레이터

오픈 타임 트립 픽업 가능 여부를 판단합니다:

1. **조건 설정:** 리포트 시간, 레그 수, 예상 블록 타임. 프리셋 버튼 또는 스테퍼 사용.
2. **결과 확인:**
   - 예상 FDP 대비 한도 (상태 뱃지).
   - 예상 릴리즈 시간 및 최대 릴리즈 시간.
   - 28일 비행시간 체크.
   - 휴식 간격 체크 (최소 10시간).
   - **최종 판정:** "Pickup OK" 또는 "Cannot Pick Up" + 경고 메시지.
3. **릴리즈 후 정보:** 최소 필요 휴식 시간 및 가장 빠른 다음 리포트 시간.

## 7. 설정

### 계정
- 이메일 확인, 로그아웃.

### 스케줄 관리
- 로스터 업로드 (.ics / .csv) 또는 Google Calendar 싱크 설정.
- 이벤트 수 확인, 스케줄 초기화.

### 유틸리티

**단위 변환기** — 항공 관련 단위 빠른 변환:
- 온도 (C / F)
- 기압 (inHg / hPa)
- 고도 (ft / m)
- 거리 (nm / km / sm)
- 속도 (kt / km/h / mph)
- 무게 (lbs / kg)

**메모** — 간단한 메모 작성. 현재 비행 레그에 자동 연결됩니다.

### 알림

- **푸시 알림:** 활성화/비활성화.
- **기상 알림:** 공항 기상 급변 시 알림 토글.
- **출근 전 리마인더:** 리포트 타임 전 1~5개 리마인더 설정 (1h, 2h, 3h 프리셋 또는 커스텀).
- **테스트 알림** 버튼으로 푸시 정상 동작 확인.

### 외형
- 테마: 라이트 / 자동 / 다크.

## 8. 팁 & FAQ

### 기상 데이터는 얼마나 자주 업데이트되나요?
METAR/TAF 데이터는 NOAA Aviation Weather Center에서 가져옵니다. 브리핑 카드를 열거나 Day 탭을 전환할 때마다 새로 가져옵니다.

### 기상 카테고리(Flight Category)란?

| 카테고리 | 운고 (Ceiling) | 시정 (Visibility) |
|----------|---------------|-------------------|
| **VFR** | > 3,000 ft | > 5 SM |
| **MVFR** | 1,000–3,000 ft | 3–5 SM |
| **IFR** | 500–1,000 ft | 1–3 SM |
| **LIFR** | < 500 ft | < 1 SM |

### 스케줄 업로드 없이도 사용할 수 있나요?
네. 브리핑 페이지에서 공항 수동 검색, 단위 변환기, FAR 117 Table B 참조는 스케줄 없이 가능합니다. 다만 대시보드, 크루, 스케줄, FDP 추적, 픽업 시뮬레이터 등 대부분의 기능은 스케줄 업로드가 필요합니다.

### 피로 위험도(Fatigue Risk)는 어떻게 작동하나요?
트립의 듀티 패턴을 분석하여 알려진 위험 요소를 감지합니다:
- **짧은 휴식** — 레이오버 11시간 미만.
- **연속 장시간 듀티** — 9시간 이상 듀티가 2일 연속.
- **늦은 릴리즈 후 이른 리포트** — 20:00 이후 릴리즈 → 06:00 이전 리포트.
- **4일 이상 연속 근무일.**

위험도: **Low** (플래그 0개), **Moderate** (1-2개), **High** (3개 이상). 이 기능은 참고용이며 공식 피로 위험 관리를 대체하지 않습니다.

### 이 앱은 공식 디스패치 도구인가요?
**아닙니다.** MFA는 파일럿의 상황 인식(Situational Awareness)을 돕기 위한 개인 참고 도구입니다. 공식 항공사 디스패치, 비행 계획, 규정 계산을 대체하지 않습니다. 항상 공식 소스를 통해 정보를 확인하세요.
